# Context

aider supports many LLM providers and exposes a model-naming
convention `provider/model-name` that maps to litellm under the
hood. EUrouter is an EU-hosted model router similar to
OpenRouter (one API, many backend models). This PR adds first-
class EUrouter support: a model-metadata cache, automatic model
selection when `EUROUTER_API_KEY` is set, model settings entries,
and docs.

The PR follows the existing `OpenRouterModelManager` pattern
almost exactly — the new `EURouterModelManager` is structurally
identical, which is the right call (consistency over novelty).
+398/-11 across 7 files.

# Diff walkthrough

**The metadata manager.** `aider/eurouter.py` is the new file:

```python
class EURouterModelManager:
    MODELS_URL = "https://api.eurouter.ai/api/v1/models"
    CACHE_TTL = 60 * 60 * 24  # 24 h

    def get_model_info(self, model: str) -> Dict:
        self._ensure_content()
        if not self.content or "data" not in self.content:
            return {}
        route = self._strip_prefix(model)
        candidates = {route}
        if ":" in route:
            candidates.add(route.split(":", 1)[0])
        record = next(
            (item for item in self.content["data"] if item.get("id") in candidates),
            None,
        )
        if not record:
            return {}
        context_len = (
            record.get("top_provider", {}).get("context_length")
            or record.get("context_length")
            or None
        )
        pricing = record.get("pricing", {})
        return {
            "max_input_tokens": context_len,
            "max_tokens": context_len,
            "max_output_tokens": context_len,
            "input_cost_per_token": _cost_per_token(pricing.get("prompt")),
            "output_cost_per_token": _cost_per_token(pricing.get("completion")),
            "litellm_provider": "eurouter",
        }
```

Two careful details: (a) the candidates set includes both the
exact route and the route with any `:suffix` stripped — this
handles `deepseek-r1:free` matching against an `id: "deepseek-r1"`
entry, which is the EUrouter pattern for variant pricing; (b)
the context-length lookup falls through `top_provider.context_length →
context_length → None`, matching how OpenRouter exposes the
field.

**The cost converter** is small but worth examining:

```python
def _cost_per_token(val: str | None) -> float | None:
    if val in (None, "", "0"):
        return 0.0 if val == "0" else None
    try:
        return float(val)
    except Exception:
        return None
```

The `"0" → 0.0` mapping is meaningful: a free model has a
genuine zero cost, distinct from "no cost data available". The
distinction matters because aider's UX shows "$0.00" for known-
free models and "(unknown cost)" for unknown ones. Conflating
them would miscolor the UI for both groups.

**The cache loading** is idempotent and tolerant:

```python
def _load_cache(self) -> None:
    if self._cache_loaded:
        return
    try:
        self.cache_dir.mkdir(parents=True, exist_ok=True)
        if self.cache_file.exists():
            cache_age = time.time() - self.cache_file.stat().st_mtime
            if cache_age < self.CACHE_TTL:
                try:
                    self.content = json.loads(self.cache_file.read_text())
                except json.JSONDecodeError:
                    self.content = None
    except OSError:
        pass
    self._cache_loaded = True
```

`OSError` from a read-only home directory (containers,
sandboxes) doesn't crash; corrupt JSON nukes the in-memory
content but doesn't raise. Both are appropriate fail-soft
behaviors for a metadata cache.

**The fetch path** writes an empty `{}` on failure:

```python
def _update_cache(self) -> None:
    try:
        response = requests.get(self.MODELS_URL, timeout=10, verify=self.verify_ssl)
        if response.status_code == 200:
            self.content = response.json()
            try:
                self.cache_file.write_text(json.dumps(self.content, indent=2))
            except OSError:
                pass
    except Exception as ex:
        print(f"Failed to fetch EUrouter model list: {ex}")
        try:
            self.cache_file.write_text("{}")
        except OSError:
            pass
```

Writing `{}` on failure is a defensible choice: it pins a
"freshness timestamp" so the next call within the TTL won't
hammer the failed endpoint again. But it loses a previously-
working cache, which is more aggressive than necessary — a
better shape would be: only overwrite if there's no existing
cache, otherwise leave the stale cache in place and print the
warning. That way a transient network failure doesn't degrade
cost/context info that was already known.

**The integration in `models.py`** follows the OpenRouter pattern
verbatim:

```python
self.eurouter_manager = EURouterModelManager()

def set_verify_ssl(self, verify_ssl):
    self.verify_ssl = verify_ssl
    if hasattr(self, "openrouter_manager"):
        self.openrouter_manager.set_verify_ssl(verify_ssl)
    if hasattr(self, "eurouter_manager"):
        self.eurouter_manager.set_verify_ssl(verify_ssl)

# In get_model_info:
if not cached_info and model.startswith("eurouter/"):
    eurouter_info = self.eurouter_manager.get_model_info(model)
    if eurouter_info:
        return eurouter_info
```

The `if not cached_info` guard means EUrouter is only consulted
when litellm's built-in metadata doesn't know the model — which
is right, because litellm may eventually add EUrouter natively
and the local manager should defer.

**Auto-selection in `onboarding.py`**:

```python
eurouter_key = os.environ.get("EUROUTER_API_KEY")
if eurouter_key:
    return "eurouter/claude-sonnet-4-5"
```

Placed before the OpenRouter check, which means if a user has
*both* keys set, EUrouter wins. That's a real opinion the PR is
taking — worth confirming with maintainers whether the order
should be reversed (OpenRouter is the more established option)
or whether having both is rare enough that the order is
arbitrary.

# Risks

1. **Cache-clobber on transient failure.** Writing `{}` on
   network error wipes a working cache. A retry-without-cache-
   clobber shape would be friendlier — leave the stale cache,
   print the warning, and try again next call.
2. **Hardcoded default model `eurouter/claude-sonnet-4-5`.**
   The model name is pinned in `onboarding.py`. If EUrouter
   renames or deprecates that model, aider's auto-selection
   silently breaks. A "find the cheapest fast Sonnet-class
   model from the cache" lookup would be more resilient, at the
   cost of complexity.
3. **No retry/backoff.** A single 10-second timeout is the
   only defense against a slow EUrouter API. For a CLI that
   may run dozens of times a day, a single retry with backoff
   would smooth out transient network blips.
4. **`requests` import.** aider already depends on `requests`
   (the OpenRouter manager uses it), so this is fine, but
   worth noting that `httpx` is the more modern choice if the
   project ever standardizes.
5. **`print(f"Failed to fetch EUrouter model list: {ex}")`** —
   prints to stdout, which may interfere with aider's output
   formatting. The OpenRouter manager probably uses the
   project's logger; a quick consistency check would be good.
6. **`EUROUTER_API_KEY` precedence over `OPENROUTER_API_KEY`**
   in onboarding is an unstated UX decision. The PR should
   probably document it in the docs page or release notes —
   users with both keys will be surprised.
7. **`@noqa: BLE001` (broad except)** appears twice. Bare
   `except Exception` is justified for "anything could go
   wrong with parsing untrusted input" but worth at least one
   reviewer eyeball to confirm no specific exceptions deserve
   distinct handling.
8. **The candidates-set `:suffix` stripping** assumes EUrouter
   only ever uses `:` as the variant separator. If they later
   add `@` or `~` for other variants (as some routers do),
   the lookup quietly stops finding them.
9. **No SSL verification override path tested.** `set_verify_ssl(False)`
   is plumbed through but the test file should exercise the
   "verify=False" path to ensure it's actually applied.
10. **Cache file format compatibility.** The cache stores the
    raw API response. If EUrouter changes their schema in a
    backwards-incompatible way, every aider user's cache becomes
    invalid until expiry. A schema version field in the cache
    file would let `_load_cache` detect mismatches and force a
    refresh.

# Suggested questions for the author

1. Is the EUrouter-before-OpenRouter precedence in
   `onboarding.py` intentional? Worth documenting either way.
2. Could the failure path *not* clobber a working cache?
   Currently a transient network failure replaces good data
   with `{}`.
3. The hardcoded `eurouter/claude-sonnet-4-5` will need
   maintenance as the EUrouter catalog evolves. Is there
   appetite for "pick best-available Sonnet" logic?
4. Should the warning message use aider's logger rather than
   `print`?
5. Are there test cases for `:suffix` model variants and for
   models missing from the catalog?

# Verdict

**Approve.** This is a well-scoped provider integration that
follows the existing OpenRouter shape — the right call for
keeping the codebase consistent. The metadata manager is
cleanly factored, the cache TTL and fail-soft behavior are
appropriate, and the integration points in `models.py` and
`onboarding.py` are minimal. The most important nit is the
cache-clobber on failure (a working cache shouldn't be wiped
because of a transient network error) and the auto-selection
precedence over OpenRouter (an unstated opinion).

# What I learned

Two takeaways, both about the shape of provider-integration code.

**First**, the *pattern-matching-existing-integration* discipline.
When a codebase has a working OpenRouter manager and you need to
add an EUrouter manager, the best PR is the one that copies the
shape exactly. New abstractions invented for the second instance
are almost always premature; you don't know which pieces will
generalize and which are specific until you have at least three
implementations. The PR copies the manager class structure, the
cache-with-TTL pattern, the `set_verify_ssl` plumbing, the
`if model.startswith("eurouter/")` dispatch in `models.py` —
verbatim. The cost is a tiny amount of duplication; the benefit
is reviewer comprehension (anyone who knows OpenRouter
immediately knows EUrouter), grep-ability ("how does X handle Y"
returns matching answers), and bug-fix uniformity (a fix to one
applies obviously to the other). Resist the urge to factor the
pattern into a base class until you have a third instance and
the abstraction is forced.

**Second**, the *fail-soft cache* idiom is a hallmark of mature
CLI tooling. Every cache-related operation in this PR is wrapped
in a try/except that swallows OSError and JSONDecodeError. A
read-only home directory doesn't crash aider; a corrupt cache
doesn't crash aider; a network failure doesn't crash aider. The
cost is some "if errors didn't happen, what would be in the
cache?" cognitive load when reading the code; the benefit is
that aider runs in containers, in CI, on locked-down corporate
laptops, on flaky wifi, and produces a sensible result every
time. The shape: every external-state operation gets a try with
a specific exception class, and the catch arm sets the in-memory
state to a known-good null/empty value. *Never* let an external
failure crash a CLI startup path.

The third meta-lesson: provider-integration PRs are a great
window into a project's engineering discipline. This one shows
healthy hygiene — pattern matching, fail-soft cache, distinct
"unknown" vs "known-zero" cost handling, candidates-set lookup
for variant pricing. The few rough edges (cache-clobber on
failure, hardcoded default model, opaque precedence over
OpenRouter) are addressable in 20-line follow-up PRs and
shouldn't block landing.
