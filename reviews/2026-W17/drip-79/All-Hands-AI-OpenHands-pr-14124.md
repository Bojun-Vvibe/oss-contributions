## What it does

Migrates the OpenHands frontend from a self-contained Vite/React app to a thin wrapper around the new shared `@openhands/agent-server-gui` package (validated against commit `2cbc504`). The bulk of `frontend/src/{api,components,routes,assets,hooks,stores}` is **deleted** and re-imported from the shared package; OpenHands-local SaaS/cloud surface (auth, org, billing, onboarding, invitations, API keys, user-settings) is what remains in the OH repo.

Despite the PR body framing this as "restore," the actual diff is overwhelmingly a *removal* — 719 deleted files, 51k lines stripped, ~9 new files added. The framing makes sense if you read "restore" as "restore SaaS-only modules **after** adopting the OSS-only shared package," but reviewers approaching the diff cold will find this confusing.

## Risk surface

**High** — this is a fundamental architecture change, not a refactor.

1. **Reviewability.** 100 files surfaced in the API but the actual diff is 778+ files. A maintainer cannot meaningfully review 21k lines of regenerated lockfile + 719 deletions + 50 rewrites in one PR. Standard practice for a migration like this is a stack of small PRs (move auth → ✓, move chat → ✓, ...) culminating in the dependency adoption.
2. **PR title says "Prototype" but the PR body presents a complete switchover.** "Prototype" implies "throwaway exploration"; merging a prototype to main locks in the shared-package architecture before either the shared package or this local override pattern have stabilized.
3. **Validation against a moving target.** Pinned to `agent-server-gui` commit `2cbc504`. If the package is in active development, every shared-package PR potentially regresses the OpenHands wrapper. The PR doesn't describe how the two repos will be co-versioned (semver? commit pin? release tag?).
4. **Human test checkbox is unchecked.** The PR body shows `[ ] A human has tested these changes.` and the QA artifacts in `.pr/qa/agent-server-gui-manual/` are screenshots rather than asserted-in-CI tests. The 10-test list in "How to Test" is the entire automated coverage for this migration.
5. **AI-agent-authored.** The PR body discloses `_This PR update was generated by an AI agent ([ide-assistant-A]) on behalf of the user._` The 13k-line lockfile regen and 50 route rewrites in particular deserve a human walking through each shim manually before merge.
6. **Loss of git history.** Deleting and re-importing 700 files breaks `git blame` for the entire frontend. If the team relies on blame to triage regressions, this should at minimum be done as `git mv` where possible (the diff shows `removed` not `renamed`, suggesting blame is gone).
7. **`api/README.md` deletion with no replacement.** Documentation for the API client layer is just removed. New contributors will need to look in the shared package, but there's no pointer left behind.

## Recommendations before this can land

1. **Split the PR.** At minimum: (a) add the `agent-server-gui` dependency + Vite resolver config, (b) migrate the API layer, (c) migrate components, (d) migrate routes, (e) finalize SaaS-only restorations. Each PR is reviewable in under an hour.
2. **Document the co-versioning policy** between OpenHands and `agent-server-gui` in `AGENTS.md` — pin format, upgrade cadence, breaking-change protocol.
3. **Add a CI job** that builds and runs the listed test suite against the pinned shared-package commit. Without this, the shared package can drift silently.
4. **Have a human run through the QA checklist** (uncheck the box in the PR body until that's done) and capture results in the PR description. Screenshots in `.pr/qa/` are not enough on their own.
5. **Replace `api/README.md`** with a one-paragraph pointer to the shared-package docs.
6. **Rename "Prototype"** in the title — either commit to landing it as the architecture or keep it on a feature branch.

Verdict: needs-discussion — the technical direction may well be right, but the PR is unmergeable in its current form due to size, missing co-versioning policy, and untested AI-generated rewrite of 50+ routes/components. Convert to a tracking issue + stacked PRs.
