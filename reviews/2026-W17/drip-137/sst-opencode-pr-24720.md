## Verdict: merge-after-nits

## Rationale

- **Right diagnosis for a real 100% CPU symptom across four independent root causes**, all stitched into one PR because they jointly produce the user-visible "fan spins on idle" report. (1) Frontend SSE reconnect tight-loop: pre-diff at `global-sdk.tsx:198-201` did `await wait(RECONNECT_DELAY_MS)` unconditionally between attempts — when WebKitGTK silently drops the IPC socket on KDE/Wayland compositor glitches, this becomes an `epoll_pwait2` loop firing every `RECONNECT_DELAY_MS`. (2) `@parcel/watcher` recursively `inotify_add_watch`-ing the entire `$HOME` when `opencode` is launched from there. (3) `IconDiscovery` glob walking `~/.cache`, `~/.mozilla`, etc. (4) Nested `node_modules` not ignored by `@parcel/watcher` because the existing `FOLDERS` patterns weren't `**/`-prefixed.
- **Backoff implementation at `global-sdk.tsx:130-208` is correct.** New `let reconnectDelay = RECONNECT_DELAY_MS` and `const MAX_RECONNECT_DELAY_MS = 10000` bounded at 10s, doubled at `:206` `reconnectDelay = Math.min(reconnectDelay * 2, MAX_RECONNECT_DELAY_MS)` after each `await wait(reconnectDelay)`, and the reset point at `:158` `reconnectDelay = RECONNECT_DELAY_MS` is correctly placed *after* `events.stream` connects but *before* the for-await loop begins, so a flapping connection that briefly succeeds then drops resets the backoff (which is the right policy — flapping should not exponentially back off forever).
- **`watcher.ts:125-130` home/root short-circuit is the right shape.** `Instance.directory === os.homedir()` and `=== "/"` cover the two pathological inputs; logged at `info` level naming the directory so the operator sees "skipping watcher for home/root directory" in logs and knows why incremental file events aren't firing. The 10K-line-cost of recursive `inotify_add_watch` on `$HOME` was the load-bearing offender so this single guard is the highest-value fix.
- **`ignore.ts:55-60` PATTERNS expansion is the missing-symmetry fix.** Pre-diff `PATTERNS = [...FILES, ...FOLDERS]` only matched top-level folders; nested `node_modules` deep in monorepos slipped through. The new triple-spread `[...FILES, ...Array.from(FOLDERS), ...Array.from(FOLDERS).map((f) => '**/${f}'), ...Array.from(FOLDERS).map((f) => '**/${f}/**')]` covers (top-level folder, nested folder name, nested folder contents) — three cells are exactly what `@parcel/watcher`'s `ignore` glob semantics require to short-circuit at every level.
- **`glob.ts:11-23` API addition + `project.ts:359-360` consumer is internally consistent.** New `ignore?: string | string[]` and `maxDepth?: number` fields on `Glob.Options` pass through to `toGlobOptions` unchanged; `project.ts` IconDiscovery now passes `ignore: ["**/node_modules/**", "**/.git/**", "**/.cache/**", "**/dist/**", "**/build/**"]` and `maxDepth: 3` — the `maxDepth: 3` is an aggressive cap that assumes icon files live within 3 dirs of the worktree root, which is true for the canonical "icon at repo root or one subdir down" case but cuts off any deep monorepo where each package has its own icon. Worth a comment naming the 3-level assumption.

## Nits / follow-ups

- The **5-file blast radius is the biggest review concern**. Each of the four root causes is independently correct but stitching them as one PR means a regression in any single file requires reverting the whole bundle. Worth splitting into (a) frontend exponential backoff, (b) home-dir watcher carve-out + nested-folder ignore patterns, (c) IconDiscovery glob options + maxDepth cap. The body acknowledges this is "Resolves multiple issues" but the linked issue is one — three follow-up PRs is the cleaner shape.
- **No test for any of the four fixes.** Backoff: no test that two consecutive failures produce `RECONNECT_DELAY_MS` then `2 * RECONNECT_DELAY_MS` waits, no test that a successful connect resets to `RECONNECT_DELAY_MS`. Watcher: no test that `os.homedir()` short-circuits return early. IconDiscovery: no test that `maxDepth: 3` actually clamps. Ignore patterns: no test that a fixture path `a/b/node_modules/x` is matched by `**/node_modules/**`. The `ignore.ts` table is the easiest to add a regression cell for — a one-liner against `match("a/b/node_modules/x", PATTERNS)`.
- **`MAX_RECONNECT_DELAY_MS = 10000` is a magic number** declared inline at `:130`. Worth promoting to a module-level `const` next to `RECONNECT_DELAY_MS` so future maintainers can find both knobs in one grep.
- **`maxDepth: 3` and the IconDiscovery hardcoded ignore list at `project.ts:359` are duplicative of `FileIgnore.PATTERNS`.** Worth either using `FileIgnore.PATTERNS` directly (DRY) or documenting why IconDiscovery uses a smaller subset (probably for perf).
- **No instrumentation** to confirm the backoff actually fires in production — a single `log.warn` at the doubling site naming the new delay would let operators verify "the loop is no longer hot" from logs alone.

## What I learned

"100% CPU on idle" is the canonical symptom shape for unbounded retry loops in `await`-based event streams, and the cure is always exponential backoff with a reset-on-success point and a `Math.min(..., MAX)` cap. The reset point is the subtle part: too-eagerly resetting (e.g. on every `for await` iteration) makes flapping connections perform like steady ones; too-late resetting (e.g. only after N consecutive successful events) makes the first reconnect after a long disconnect take forever. The "right after the stream subscribe call succeeds" point used here is the canonical answer. Separately, the `@parcel/watcher` ignore-pattern expansion to three cells (`f`, `**/f`, `**/f/**`) is the load-bearing recipe for any glob-based ignore table that needs to match folders at any depth — a pattern that should probably be encapsulated in a `nestedFolderPatterns(name: string): string[]` helper to prevent drift between watch and glob consumers.
