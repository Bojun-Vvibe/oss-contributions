## Specifics

- The build-tag split is the right shape: `native.go` becomes `native_other.go` with `//go:build !darwin` (`native_other.go:1`), and the new `native_darwin.go:1` carries `//go:build darwin`. Both define `NativeBackend` with the same external API (`NewNativeBackend(icon any) *NativeBackend`, `Send(n Notification) error`, `SetNotifyFunc`/`ResetNotifyFunc`), so callers compile against the same surface on either platform.
- `terminalBundleIDs` map at `native_darwin.go:14-23` maps `TERM_PROGRAM` env values to bundle IDs for Apple_Terminal, ghostty, iTerm.app, WezTerm, kitty, Alacritty, vscode. `tmux` and `screen` are explicitly mapped to `""` (`:21-22`) with the comment "runs inside another terminal" — that's the correct shortcut: when running under tmux the activation target should be the *outer* terminal, which `TERM_PROGRAM` won't reveal. The current code falls through to "no activation" in that case rather than trying to guess; reasonable.
- `lookupBundleID` at `native_darwin.go:74-87` is a 2s-bounded osascript fallback (`id of app "<name>"`) for terminals not in the static map. It tries both `appName` and `strings.TrimSuffix(appName, ".app")` (`:77`). Fine. The 2s timeout on a per-startup osascript exec is conservative — most lookups will be sub-100ms — but it's a one-shot at backend construction so total cost is bounded.
- `Send` at `native_darwin.go:92-109` short-circuits on `b.hasTerminalNotifier`; otherwise falls through to the prior `beeep.Notify` path. So this is a *progressive enhancement* — users without `terminal-notifier` installed get unchanged behavior. README at `:472-487` documents the `brew install terminal-notifier` step.
- `sendWithTerminalNotifier` at `native_darwin.go:112-138` shells out to `terminal-notifier -title <t> -message <m> -group crush` plus optional `-activate <bundleID>`. The `-group crush` flag enables coalescing/replacement (subsequent notifications with the same group ID replace the prior one in Notification Center) — good UX. 5s timeout on the exec via `context.WithTimeout` (`:122-123`) is sane.

## Concerns

- `terminalBundleIDs` hardcodes a snapshot of TERM_PROGRAM values that's already incomplete — common ones missing: `WarpTerminal` (Warp), `Hyper`, `Tabby`, `rio`, plus the various JetBrains terminals which set `TERM_PROGRAM=JetBrains-JediTerm`. The `lookupBundleID` osascript fallback covers some of these (e.g. `id of app "Warp"`) but not the JetBrains case where `TERM_PROGRAM` doesn't match an app name. Not a blocker, but worth a TODO comment or a hint that PRs to extend the map are welcome.
- `NewNativeBackend`'s `exec.LookPath("terminal-notifier")` (`:51-54`) and the `lookupBundleID` osascript call (`:67-69`) both run synchronously at construction time. If `NewNativeBackend` is called on the hot path during TUI startup, the osascript-fallback can add 50-200ms of startup latency on macOS for users with non-mapped terminals. Worth confirming this is constructed lazily or once-globally rather than per-notification.
- `terminal-notifier` is a third-party brew formula that hasn't seen a release since ~2019 (last tag `2.0.0`); it works on current macOS but Apple has been progressively tightening notification entitlements. A note in the README that `terminal-notifier` requires the user to grant Notification Center permission *to terminal-notifier itself* (not to crush) on first run would prevent a confusing first-time UX where notifications silently no-op until the user finds the System Settings → Notifications pane.
- No tests added. The `SetNotifyFunc`/`ResetNotifyFunc` hooks at `native_darwin.go:140-152` look designed for testing (the comment "allows replacing the fallback notification function for testing" confirms this) but no test file is included that exercises them. At minimum a `TestNativeBackend_FallsBackToBeeepWhenNoTerminalNotifier` would lock down the LookPath-fail branch.
- The static map keys use `TERM_PROGRAM` strings as observed (e.g. `"Apple_Terminal"`, `"iTerm.app"`, `"WezTerm"`) but the casing is inconsistent across the ecosystem — `ghostty` is lowercase, `iTerm.app` has the `.app` suffix, `Alacritty` is title-case. That's just how those terminals chose to set the env var; the map matches reality. Worth a comment that these are exact `$TERM_PROGRAM` values, not casing-normalized lookups, so a future contributor doesn't try to "fix" them.

## Verdict

`merge-after-nits` — the build-tag split is clean, the static map + osascript fallback is the right shape for the long tail, and the progressive-enhancement design (no behavior change without `terminal-notifier` installed) means low merge risk. Two real follow-ups: (1) confirm `NewNativeBackend` isn't called on the hot path so the osascript-fallback latency doesn't bite, and (2) add even one test for the LookPath-fail-→-beeep fallback branch. The missing TERM_PROGRAM entries (Warp, JetBrains, Hyper) are fine to land as a follow-up PR.
