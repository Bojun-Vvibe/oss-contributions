## Rationale

This PR collapses the existing `packages/desktop-electron` (Electron) and `packages/desktop` (Tauri) packages into a single Electron-based `packages/desktop`, deleting `src-tauri/` and the entire `desktop-electron/` directory in the process. The diff is dominated by file-move noise (icons under `packages/desktop/icons/{beta,dev}/...` showing 0/0 because they're binary moves), but the substantive changes are: deletions of `packages/desktop-electron/{package.json,tsconfig.json,scripts/*}`, the new `packages/desktop/electron-builder.config.ts` and `packages/desktop/electron.vite.config.ts`, an updated root `package.json`, and a rewritten `.github/workflows/publish.yml:1-10`.

The intent — kill the Tauri/Electron split and standardize on Electron — is reasonable and the bulk-deletion side of the diff is mechanically correct (the moved-icon entries showing identical paths under the new tree confirm a clean rename rather than re-encoding). However, this is a `needs-discussion` because (1) the PR body's own Testing checklist is entirely unchecked — "Build process works correctly", "Desktop app launches and functions properly", "CI/CD pipeline updated and working" are all `[ ]` — meaning the author has not verified the consolidated package actually builds or runs end-to-end on at least one platform; (2) the change is explicitly marked as containing breaking changes ("The desktop package no longer uses Tauri"), which means downstream consumers, packaging scripts, and any signing/notarization tooling tied to Tauri's `src-tauri/tauri.conf.json` need a coordinated migration story that the PR doesn't include; and (3) `bun.lock` shows −93/+2, which suggests Tauri Rust deps and several Tauri JS deps have been removed but only minimal Electron deps added — worth confirming the Electron toolchain (`electron-builder`, `electron-vite`, `@electron/rebuild`, native module support) is actually pulled in transitively rather than relying on hoisted versions from another package.

The 10/10 line swap in `.github/workflows/publish.yml` is the one piece of the diff worth eyeballing carefully before merge: a desktop publish pipeline that worked for Tauri's `cargo tauri build` will not transparently work for Electron's `electron-builder --publish`. Code-signing identities, entitlements files, macOS notarization stapling, and Windows Authenticode signing all live in different config keys between the two toolchains, and the publish workflow needs to be re-validated on a real release branch (or at minimum a dry-run with `--publish never`) before this lands.

Recommendation: split the discussion into two — (a) get a maintainer to confirm "we're moving off Tauri permanently" is the intended direction (this is a significant strategic shift for the desktop story), and (b) require at least one passing `npm run build && npm run start` log from each target OS attached to the PR before merge. The consolidation is the right end state, but landing it on an unchecked test list is risky given the blast radius.
