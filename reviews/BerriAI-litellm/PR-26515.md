## Summary

The router runtime accepts fallback chain entries as either a string (model name) or a dict (model name + per-call overrides like `api_key`, `extra_headers`). The dashboard's `Fallbacks` settings page only handled the string shape, so a dict entry crashed React with error #31 ("objects are not valid as a React child"). This PR teaches the UI to detect the dict shape, render the `model` field as the display label, and tag it with an `override` badge so admins can see at a glance that extra params are attached.
