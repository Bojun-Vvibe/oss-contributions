## Summary

Vertex AI's Anthropic Claude variants accept `output_config.format` (structured outputs / JSON schema) but reject `output_config.effort` with a 400 "Extra inputs are not permitted". The previous code defended by stripping the entire `output_config` object before forwarding, which silently dropped the supported `format` bits. This patch adds a leaf module `output_params_utils.py` with `sanitize_vertex_anthropic_output_params` that filters only the unsupported keys (currently `frozenset({"effort"})`) and forwards the rest. Both the chat-completion and Messages pass-through transformations now call the helper. The PR also includes a separate fix for Arize span attributes (`_safe_get` for non-dict usage objects) and unrelated additions to model price tables.

## Reasoning

The core fix (selective sanitization of `output_config` for Vertex Anthropic) is well designed and the leaf-module placement correctly addresses the cyclic import flag. But this PR bundles at least three independent changes: the Vertex `output_config` fix (the title), an Arize `usage` Pydantic-vs-dict shim (a separate referenced issue), and a batch of model-price additions plus dashboard helpers and a Cloudflare transformation tweak. Each deserves its own bisectable commit and its own focused review. The Vertex helper itself also lacks direct unit coverage (the helper's invariants are inferred from the integration tests rather than asserted directly). Split the PR, add the focused helper tests, and re-request review.
