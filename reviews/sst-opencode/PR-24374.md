## Summary

Switches both v1 and v2 SDK server entrypoints from `import launch from "cross-spawn"` to `createRequire(import.meta.url)("cross-spawn")`. `cross-spawn` ships a CommonJS-only entry; the previous ESM `default` import relied on bundler/runtime interop to synthesize a default export, which Bun's raw ESM loader does not do for plugins shipping their own SDK copy. The new form pulls the CJS export directly with no default-interop assumption.

## Verdict: `merge-as-is`

Two-line, surgical fix. The `typeof launchType` import keeps the call-site type contract identical, so no downstream code change is needed. The same fix is applied symmetrically to `packages/sdk/js/src/server.ts` and `packages/sdk/js/src/v2/server.ts`, which is correct — leaving v2 untouched would have left half of consumers broken.

## Specific references

- `packages/sdk/js/src/server.ts:1-7` — `import { createRequire } from "node:module"` + `import type launchType from "cross-spawn"` + `const launch = createRequire(import.meta.url)("cross-spawn") as typeof launchType`. The `as typeof launchType` cast is the right way to do this: the runtime value loses its TS shape after going through `createRequire`, but the type-only import re-attaches the contract without emitting a runtime import that Bun would re-evaluate.
- `packages/sdk/js/src/v2/server.ts:1-7` — identical change, mirrored. Good.

The author's verification (`bun typecheck` from `packages/sdk/js`, `bun ./script/build.ts`, and a one-liner `await import('./src/server.ts'); await import('./src/v2/server.ts')` which exercises the new `createRequire` path under Bun) is the right minimum. No new automated test is needed — there's nothing to assert beyond "the import doesn't throw under Bun's raw loader," which the build itself already proves.

## Nits (optional)

1. A one-line comment at each call-site (`// cross-spawn is CJS-only; createRequire avoids Bun's no-default-interop ESM loader. See #22281.`) would prevent a well-meaning future refactor from "modernizing" this back to `import launch from "cross-spawn"`.
2. If anyone else in the SDK does `import x from "<cjs-only-pkg>"`, this same fix likely applies. A grep for `import \w+ from "cross-spawn"` in the wider repo (or just `gh search code`) would catch parity. Out of scope for this PR though.
