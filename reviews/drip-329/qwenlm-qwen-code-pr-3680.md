## Summary

Expands the TUI markdown renderer to support Mermaid diagrams (with image-protocol fallback chain Kitty → iTerm2 → chafa, plus regex-based text wireframe fallback for ~10 mermaid families), block/inline LaTeX (Unicode substitution), task lists, and blockquotes. Adds `ui.renderMode` setting (`render` | `raw`) and `Alt/Option+M` session-level toggle. Extends `/copy` with `/copy mermaid N`, `/copy latex N`, `/copy latex inline N`.

## Notes

- `packages/cli/src/ui/utils/mermaidImageRenderer.ts` (+825) — the heart of the image path. Capability-gated by `QWEN_CODE_MERMAID_IMAGE_RENDERING=1`, auto-disabled under tmux/SSH, with `QWEN_CODE_MERMAID_IMAGE_PROTOCOL=kitty|iterm2|off` override. The default-off + opt-in env is the right call given the variability of terminal image protocol support; merging with default-on would have been a footgun.
- `packages/cli/src/ui/utils/mermaidVisualRenderer.ts` (+1,508) — single largest new file. Hand-rolled regex parsers for ~10 Mermaid diagram families (`flowchart`, `graph`, `sequenceDiagram`, `classDiagram`, `stateDiagram`, `erDiagram`, `gantt`, `pie`, `journey`, `mindmap`, `gitGraph`, `requirementDiagram`). PR body honestly disclaims this is "not a complete Mermaid layout engine" and unknown families fall back to the original fenced source. The disclosure is responsible, but 1.5k LOC of regex parsing is a meaningful long-term maintenance burden — every Mermaid release that tweaks syntax risks silent fallback regressions. Recommend an explicit "we accept fallback regressions on Mermaid syntax churn" note in the docs.
- `packages/cli/src/ui/utils/latexRenderer.ts:1-144` — Unicode-substitution LaTeX renderer. Inline `$...$` and block `$$...$$`. Supports Greek letters, basic operators, arrows, `\frac{a}{b}`, `\sqrt{x}`, limited super/subscript. Matrices, environments, macros explicitly out of scope. The "lightweight Unicode substitution" framing is correct — anything else would be vapor in 144 LOC. Worth noting: a malformed `$` (e.g., a single dollar sign in a code-adjacent context) could mis-trigger inline math; recommend a test for `$5 and $10` (currency-like) to ensure no inline-math hijacking.
- `packages/cli/src/ui/utils/MarkdownDisplay.tsx` (+331/-14) — extends the existing regex-block-detection parser rather than rewriting around an AST. PR body explicitly states "this PR does not replace the whole Markdown parser with an AST pipeline" — agree this is the right scope to keep the change reviewable; an AST migration is a separate mission.
- `packages/cli/src/ui/commands/copyCommand.ts` (+368/-2) — adds `/copy mermaid N`, `/copy latex N`, `/copy latex inline N`. Test file (+420 LOC) covers the new branches comprehensively. Sample at `copyCommand.test.ts:793-820` shows `/copy code` correctly returning the *last* fenced code block, including a Mermaid block — confirming the existing `/copy code` semantics still treat Mermaid as fenced code.
- `packages/cli/src/ui/contexts/RenderModeContext.tsx` (+25) — new React context for the session-level render mode. `AppContainer.tsx:763-781` wraps the tree with `RenderModeProvider` inside the existing `CompactModeProvider`. - `packages/cli/src/config/keyBindings.ts:+5` — registers Alt/Option+M for the toggle. Need to confirm this doesn't collide with any platform shortcut (macOS `Option+M` is `µ` in some keymaps; the keypress layer should already eat the modifier before the µ lands). `KeypressContext.test.tsx:+31` adds coverage for the new keybind.

- Risk surface this introduces: (a) Mermaid CLI / `mmdc` is a new transitive dependency for the image path (gated behind `QWEN_CODE_MERMAID_ALLOW_NPX=1` for ad-hoc), (b) terminal image protocol detection is heuristic and will mis-classify some terminal multiplexers, (c) the regex Mermaid parser is brittle to Mermaid syntax churn. All three are documented in PR body.
- Windows / Linux terminal image paths "still need reviewer or follow-up validation" per PR body. Given the fallback chain to plain text, shipping macOS-only-validated is acceptable, but should be land-then-bake on a feature flag for at least one release before defaults change.

Probably too late to ask for the split now.
