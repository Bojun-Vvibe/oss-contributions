# BerriAI/litellm #27040 — fix(bedrock): forward output_config.effort for adaptive-thinking Claude

- **Head SHA:** `2a2fcb32a63b97ccb812f1cd1770fdfe09cf1e94`

- **Verdict:** `merge-after-nits`

## Rationale

Plugs a real wire-shape bug for Claude 4.6/4.7 adaptive-thinking on Bedrock Converse. Anthropic's Messages API takes `output_config.effort`; AWS Bedrock Converse takes the same value as `additionalModelRequestFields.thinking.effort`. Callers chaining a Messages-shaped body through Converse were having `output_config` stripped before the value reached the wire (the existing `inference_params.pop("output_config", None)` at `converse_transformation.py:~1283`), so requested effort was silently dropped. The fix adds `_fold_output_config_effort_into_thinking` (`converse_transformation.py:482`) which gates on `AnthropicModelInfo._is_adaptive_thinking_model(model)`, only injects `thinking.effort` when the caller hasn't already set one, and runs *before* the `output_config` pop. The companion change in `_handle_reasoning_effort_parameter` (`converse_transformation.py:455`) maps `reasoning_effort` symbolically (`low/medium/high/xhigh/max`) into `thinking.effort` for the same model class, with a passthrough for non-symbolic values.

Nits: (1) `effort_map` at `converse_transformation.py:464` aliases `minimal → low` — verify with the AWS adaptive-thinking docs that "minimal" should not just pass through; the comment doesn't justify the collapse, and a future Anthropic addition like `effort: minimal` would now silently degrade; (2) flipping `effort-2025-11-24` from `null` to `"effort-2025-11-24"` in `anthropic_beta_headers_config.json` ships the beta header on every request that opts into effort — confirm AWS Bedrock is OK with this beta header (Bedrock historically rejects unknown anthropic-beta headers in `additionalModelRequestFields`); (3) the `_translate_legacy_thinking_for_adaptive_model` flow is referenced in the new docstring but the helper here doesn't call it — add a unit test that exercises the case where a caller passes both `thinking: {type: enabled}` and `output_config.effort` to confirm the precedence order is what the docstring claims.

Risk is medium because the change touches three transformation entrypoints (Converse, plus invoke and messages variants of `anthropic_claude3_transformation.py`) and ships a beta header flip. The new tests across all three transformation paths are the right level of coverage; merge after the `minimal` aliasing and beta-header questions are answered.
