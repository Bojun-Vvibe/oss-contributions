## Concerns / nits

1. **Defense-in-depth: the guards in `handleItemToggle`/`handleEditCommit`/`handleItemClear` rely on `writableSelectedScope` being kept in sync.** If `editableScopeItems` is empty, `writableSelectedScope` is `undefined` and writes are suppressed — good. But the underlying `setSetting` in `LoadedSettings` does not appear (in this diff slice) to itself check `readOnly` before mutating. A read-only scope could still be written if a future caller bypasses the dialog. Worth a follow-up to add a `readOnly` guard at the `setSetting`/`setValue` layer in `settings.ts` so the invariant is enforced at the data boundary, not just in the UI.
2. **`useEffect` at `:125-128` will trigger an extra render** on dialog mount when the persisted `selectedScope` doesn't match the editable list. That's correct behavior, but the user might briefly see the wrong scope highlighted. Worth either initializing `useState<LoadableSettingScope>(writableSelectedScope ?? SettingScope.User)` directly to avoid the post-mount re-render flicker, or accepting the small flicker as cost of the fix.
3. **The Workspace-path-undefined carve-out** at `:115-118` is not separately tested. The new test at `:611` configures `system: createSettingsFile('', {}, true)` (read-only) and the old `should not offer the home-directory workspace` test at `:625-` uses `workspace: createSettingsFile('', {}, true)` — it conflates "workspace path is empty string" with "workspace is read-only", and the filter predicate has both terms. A test that sets `workspace.path === undefined` *but* `readOnly !== true` (i.e., the home-directory case the carve-out was originally designed to handle) would lock that branch independently.
4. **`getSnapshot()` mutation tests rely on the existing `loadedSettings` fixture.** The new `getSnapshot() should preserve readOnly metadata` test constructs a fresh `LoadedSettings` directly. Worth confirming this fresh construction is the canonical pattern (vs using the existing fixture and overriding `readOnly`) — the file already has many tests that use the fixture, so the fresh construction is unusual and may drift from the fixture's invariants over time.
