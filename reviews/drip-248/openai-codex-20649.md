# openai/codex #20649 — [connectors] Track host-owned Codex Apps MCP provenance

- Link: https://github.com/openai/codex/pull/20649
- Head SHA: `6b3013938cb3d9020a2e13b5db25391cac33369b`
- Author: mzeng-openai

## Summary
Closes a trust-boundary hole: the connector-gateway pre-PR identified the host-injected `codex_apps` MCP server by **server name string match**. Server names are routing labels, not ownership boundaries — a user-configured MCP entry could legitimately be named `codex_apps` and would then transparently inherit connector-gateway behavior (runtime auth, cache contexts, special tool filtering). Fix introduces a runtime-only `McpServerProvenance` enum with two variants (`UserConfigured` default, `HostOwnedCodexApps`), threads it through `McpServerConfig`, `AsyncManagedClient`, and `ToolInfo`, and replaces every raw-name check across the connector layer with a typed provenance equality check. +380/-93 across 34 files.

## Line-level observations
- `codex-rs/config/src/mcp_types.rs:39-46` — new enum is `#[derive(Debug, Clone, Copy, Default, PartialEq, Eq)] pub enum McpServerProvenance { #[default] UserConfigured, HostOwnedCodexApps }`. The `Copy` + `Default` + `Eq` derives are exactly right: it's a 1-byte tag, callers can treat it as a value, and the `#[default] UserConfigured` ensures any deserialized config (which uses `#[serde(skip)]` on the new field at `:150-152`) gets the safe-by-default value. The fail-closed default is the load-bearing decision — a forgotten initialization gives the user-trust value, not the host-trust value.
- `codex-rs/config/src/mcp_types.rs:347` and matching `RawMcpServerConfig::TryFrom` arm explicitly stamps `provenance: McpServerProvenance::UserConfigured` on every config that arrives via deserialization (TOML, JSON, env). The only place that can mint `HostOwnedCodexApps` is the host's own server-config builder at `codex-mcp/src/mcp/mod.rs` (function `codex_apps_mcp_server_config`, see test at `mod_tests.rs:228-241` which asserts `codex_apps_mcp_server_config(&config).provenance == McpServerProvenance::HostOwnedCodexApps`). This restricts the trusted-tag mint site to a single function — exactly the right shape for a privilege boundary.
- `codex-rs/codex-mcp/src/connection_manager.rs:120,131,193` — every prior `server_name == "codex_apps"` style check (across cache setup, gateway routing, and tool filtering) is replaced with `client.provenance == McpServerProvenance::HostOwnedCodexApps`. The `.find(|(_, client)| client.provenance == ...)` pattern at `:193` correctly returns `None` when no host-owned server exists, instead of falling through to a misnamed user server.
- `codex-rs/codex-mcp/src/rmcp_client.rs:472,480-483` — `AsyncManagedClient` gains `pub(crate) provenance: McpServerProvenance` and `start_server_task` now takes `provenance: McpServerProvenance` as an explicit parameter (rather than `Option<&str>` of a name) at `:480`, computing `let is_host_owned_codex_apps_server = provenance == McpServerProvenance::HostOwnedCodexApps;` so the runtime auth/discovery branch is gated typed-not-stringly. The `mcp/auth.rs:138` runtime-auth gate also flips to `config.provenance == McpServerProvenance::HostOwnedCodexApps`.
- `codex-rs/codex-mcp/src/tools.rs` and `core/src/mcp_tool_call.rs:1051` — the `ToolInfo` struct gains `pub server_provenance: McpServerProvenance` and a helper `fn is_host_owned_codex_apps(&self) -> bool` at `:694` so downstream consumers (skill-dependency analysis, tool-call dispatch) check provenance instead of re-parsing the namespaced tool name. Tests at `connection_manager_tests.rs:307-360` (`host_owned_codex_apps_server_is_identified_by_client_provenance`) cover the dispositive case where a `UserConfigured` client *named* `codex_apps` is correctly NOT identified as host-owned.
- The `McpServerConfig` field is `#[serde(skip)]` at `mcp_types.rs:150-152` so the provenance never round-trips through the wire — clients cannot forge it via a crafted config payload, and the value is reset to `UserConfigured` on every deserialization. This is the correct serialization stance for a runtime-only trust marker.

## Verdict
`merge-as-is` — textbook fix for a "string was load-bearing for a privilege boundary" bug. The new `McpServerProvenance` puts the trust boundary at a typed enum with a fail-safe `Default`, a single mint site, `#[serde(skip)]` so it can't be forged through deserialization, and a regression test that exercises the dispositive case (user-configured server *named* `codex_apps` is correctly treated as user-trust). The 34-file blast radius is the cost of doing it right — every prior name check that was load-bearing now goes through the typed gate.
