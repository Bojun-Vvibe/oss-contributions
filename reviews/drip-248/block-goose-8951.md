## Summary
Cleanup PR landing two days after #8950 (settings-modal decomposition). Three coordinated edits: (1) remove redundant per-tab `description` props from 8 `SettingsPage` invocations now that the shared header line is in place upstream — this is the load-bearing change since the decomposed sections were duplicating header copy; (2) flesh out `AboutSettings.tsx` with a real metadata table sourced from `@tauri-apps/api/app` (name/version/build-mode/Tauri-version/identifier/license) replacing the prior empty-shell `description="View app version..."`; (3) refine compaction-tab copy in 7 locale JSON files (`en`, `es`, plus 5 others not visible in the slice). +135/-70 across 12 files.

## Line-level observations
- `ui/goose2/src/features/settings/ui/AboutSettings.tsx:42-77` — new `useEffect` block dynamically `import("@tauri-apps/api/app")` only when `window.__TAURI_INTERNALS__` exists, then `Promise.all([getName(), getVersion(), getTauriVersion(), getIdentifier()])` and a `cancelled` flag in the cleanup function to prevent state updates after unmount. The dynamic import + `__TAURI_INTERNALS__` guard is the correct shape for code that ships in both Tauri and browser-test contexts (the storybook/vitest harness has no Tauri runtime). The try/catch sets `appInfo` to `null` on failure so the component degrades to "Unavailable" labels rather than crashing — fail-safe.
- `AboutSettings.tsx:88-114` — the new `AboutInfoRow` presentational sub-component (extracted at top of file) renders one label/value pair with `truncate text-right text-sm font-medium` on the value. The 6 rendered rows use `appInfo?.<field> ?? fallback` so partial data shows what's available rather than blanking everything. The `import.meta.env.DEV` build-mode row is the right way to surface dev-vs-prod since Vite resolves it at build time (no runtime probe). The hardcoded `"Apache-2.0"` license string at `:113` is fine for now but worth pulling from `package.json` or a build-time constant if the license ever changes.
- `ui/goose2/src/features/{extensions,settings/{Chats,Compaction,General,Projects,VoiceInput}}Settings.tsx` plus `ExtensionsSettings.tsx:8` — every `<SettingsPage description={t(...)} />` call drops the `description` prop, paired with the removal of the corresponding key in `shared/i18n/locales/{en,es,...}/settings.json` (e.g. `:296` removes `"description": "View app version, build info, and licenses."` from the en bundle, `:393` removes the es equivalent). The translation-bundle cleanup is the right hygiene — leaving orphaned keys would silently bloat the bundle. Worth confirming all 7+ locales are updated (only `en`/`es` visible in the slice; `de`/`fr`/`ja`/`zh`/`pt`/`ko` etc. need the same removal).
- `GooseAutoCompactSettings.tsx:198-200` and `settings.json:329,338-339` — compaction copy is rewritten from "Choose when Goose should compact older context before replying. This applies to all Goose chats." to "Choose when the Goose agent compacts older context during sessions. This applies to all goose chats." The copy refinement is fine but the lowercased `"goose chats"` in the new string is inconsistent with the `"Goose"`-capitalized brand name elsewhere — likely a typo, should be `"Goose chats"`.
- `DoctorSettings.tsx:153,161` — `variant="outline"` applied to two action buttons aligning their color with the standard button style, as called out in the PR body. No behavior change, pure visual normalization.
- The PR runs `pnpm --dir ui/goose2 check`, `typecheck`, and a targeted vitest for `GooseAutoCompactSettings.test.tsx`. There's no test added for the new `AboutSettings` Tauri-data flow — a smoke test asserting that with `__TAURI_INTERNALS__ === undefined` the component renders fallback labels (and doesn't throw) would lock the browser-context degradation path.

## Verdict
`merge-after-nits` — fix the `"goose chats"` lowercase inconsistency in the new compaction copy (en + es + other locales), add a smoke test for `AboutSettings` rendering in the no-Tauri-runtime case, and confirm the orphaned `description` keys are removed from all locale JSON bundles (not just `en`/`es`). The Tauri data flow (dynamic import + `__TAURI_INTERNALS__` guard + `cancelled` flag + try/catch fallback) is exactly the correct shape for hybrid Tauri/browser-test code, and the redundant-description cleanup is the right follow-through after #8950 introduced the shared header.
