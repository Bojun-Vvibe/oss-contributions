## Summary

Changes `InferenceRuntime::get_or_init` from `-> Arc<Self>` (with an internal `panic!` on `LlamaBackend::init` failure) to `-> Result<Arc<Self>>`, propagating the error to both call-sites (`AppState::new` in `goose-server/src/state.rs:51` and `LocalInferenceProvider::from_env` in `local_inference.rs:360`).

## Verdict: `merge-as-is`

Tiny, mechanical, correct refactor that removes a panic from the server startup path. `LlamaBackend::init` failing is a perfectly recoverable condition (missing GPU drivers, conflicting CUDA versions, OOM at runtime-init time) and the panic was forcing the entire goose-server process down for what should be a per-feature degradation.

## Specific references

- `crates/goose/src/providers/local_inference.rs:65-68` — the `Weak::upgrade()` early-return is now `Ok(runtime)`. Correct: if a previous successful init produced a still-live `Arc`, we return it without re-running the backend init.
- `crates/goose/src/providers/local_inference.rs:83` — `Err(e) => return Err(anyhow::anyhow!("Failed to initialize llama backend: {}", e))`. The error message preserves the underlying cause via `{}`, so downstream `tracing::error!` will still surface the root reason. Good.
- `crates/goose/src/providers/local_inference.rs:91` — the final `Ok(runtime)` reuses the same `Arc` that `*guard = Arc::downgrade(&runtime)` registered on, so the singleton invariant is preserved across the success path. The Safety-invariant comment immediately above (preserved in the diff context) is still accurate post-change.
- `crates/goose-server/src/state.rs:51` — `InferenceRuntime::get_or_init()?` correctly propagates with `?` because `AppState::new` already returns `Result`. No additional plumbing needed at this call-site, which is the cleanest possible outcome.
- `crates/goose/src/providers/local_inference.rs:360` — `LocalInferenceProvider::from_env` already returns `Result<Self>` so the `?` is again free.

The remaining `panic!` in this file (line ~78, `"runtime should be uninitialised here; the mutex guard prevents concurrent re-init"`) is correctly **not** converted, because that one is a true logic-invariant violation (it would only fire if the `Weak::upgrade()` lied), not a recoverable runtime condition. Right call to leave it.
