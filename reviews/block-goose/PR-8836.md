# block/goose PR #8836 — drop -i flag from login shell PATH resolution to prevent SIGTTOU

## Summary

Removes the `-i` (interactive) flag from the helper that spawns the user's login shell to read `PATH`. An interactive shell can call `tcsetpgrp()` to claim the foreground process group of the controlling terminal; when goose later switches into raw mode (rustyline) it finds itself in a background pgroup and is suspended by SIGTTOU. Switching to `-l` only is sufficient to source the profile files where users typically set `PATH`, and removes the pgroup-stealing window.

## Verdict: `merge-as-is`

The fix is exactly the right scope: change two call sites, both of which only run `echo $PATH`, neither of which needs an interactive shell. The added comments explain the *why* (SIGTTOU mechanism) which is more valuable than any test could be here, since reproducing the race deterministically would require a controlling-terminal harness.

## Specific references

- `crates/goose-mcp/src/subprocess.rs:60-66` — first call site. The block comment explicitly names `tcsetpgrp()` and SIGTTOU which is the right level of detail; future maintainers won't be tempted to "restore" `-i` for completeness.
- `crates/goose/src/agents/platform_extensions/developer/shell.rs:171-187` — second call site, both the flatpak and direct branches are updated symmetrically. Important: the `flatpak_spawn_process` arm passes the shell name as a positional argument followed by `["-l", "-c", ...]`, which is correct for the `flatpak-spawn --host` invocation pattern.

## Nits

1. The two block comments are nearly identical. Consider hoisting the resolver into a shared util in `goose-utils` (or wherever cross-crate helpers live) so the rationale lives in exactly one place and the two crates can't drift.
2. Some users source `PATH` only in `.bashrc` (interactive rc) rather than `.bash_profile` (login). Worth a brief note in `CHANGELOG`/migration docs that if a user's `PATH` regresses on this version, the right fix is to move `export PATH=...` out of `.bashrc` and into `.bash_profile` / `.zprofile`.
3. Could swap the `Command::new(&shell).args([...])` for a slice constant to keep the two branches and the goose-mcp copy literally identical at the syntactic level — easier to grep for future changes.
